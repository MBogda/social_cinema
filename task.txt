Тестовое задание

Система бронирования билетов в социальном кинотеатре. 
Просмотр информации и покупка билетов.

Считаем, что пользователь предварительно прошел регистрацию, авторизацию и т.п., 
например по номеру телефона или почте.

Две роли пользователя:
1. Клиент.
2. Администратор.

Клиенты могут быть следующих категорий:
1. Обычная.
2. Социальная-1 (x% скидка).
3. Социальная-2 (y% скидка).
4. Социальная-3 (z% скидка).

Сервис позволяет клиенту как минимум:
1. Просмотреть список сеансов.
2. Забронировать билет на сеанс.
3. Просмотреть информацию о забронированных местах.
4. Отменить бронирование.
5. Просмотреть историю посещенных сеансов.

Сервис позволяет администратору как минимум:
1. Добавить сеанс.
2. Перенести сеанс.
3. Отменить сеанс.

Социальная категория имеет приоритет в покупке билетов.
Чем больше индекс социальной категории, тем больше скидка на цену билета.

По-умолчанию, обычная категория может забронировать билет только за m минут до начала сеанса.

Должна быть возможность отключения социальных льгот, 
то есть уравнивания категорий граждан между собой.

Если заполняемость зала ниже определенного значения, 
возможность покупки билета для обычной категории может открыться раньше.
Владелец кинотеатра стремится получить не меньше определенного % прибыли при полной посадке на сеанс.

Не полный список примеров:
1. Скорость заполняемости зала = min, в течении n минут. Принимается решение открыть продажу обычной категории.
2. Динамика заполняемости зала = max. Если так пойдет и дальше, весь зал однозначно будет заполнен льготной категорией. Принимается решение открыть продажу обычной категории.

Требования к реализации:
1. Исходный код на Github.
2. Реализован на Kotlin (jdk 8).
3. Должен собираться, запускаться, иметь тестовые данные.
4. Не должны требоваться установка отдельных программ для запуска.
5. Должен быть описан README.MD.
6. Коммиты должны быть на русском языке.
